Week 08 - Weighted Graphs and Dijkstra

This page focuses on one core difference: BFS counts edges, but Dijkstra counts total cost. Use the tabs to inspect weighted path ideas, step through Dijkstra, and reconstruct a shortest path.

Key Ideas

Dijkstra is not just “BFS with numbers.” It tracks the cheapest known total cost at every step.

Weighted Edge

Every edge has a cost. A path cost is the sum of those edge weights.

Tentative Distance

Each node stores the best known cost from the start so far. The value can improve later.

Relaxation

Check whether going through the current node gives a smaller distance to a neighbor.

Settled Node

When the smallest tentative node is chosen, its shortest distance is fixed.

Dijkstra Lab

The same graph supports all three views, so concept, algorithm, and final path stay aligned.
